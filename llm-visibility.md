# LLM Visibility

## Scope

This file covers extractability, citations, entity clarity, and trust signals from the report.

## Rules

### 1) Make content easy to extract into passages
- **Rule name:** Extractable passages
- **Description:** Use semantically coherent chunks so models can pull a useful passage cleanly.
- **Source URL:** https://www.lumar.io/blog/best-practice/4-pillar-geo-strategy-framework-for-ai-search-visibility/
- **Source organization:** Lumar
- **Source type:** Industry analysis
- **Classification:** Emerging GEO Practice
- **Severity:** Opportunity
- **Confidence level:** Medium
- **Evidence strength:** Moderate
- **Why it matters:** The report argues retrieval happens at passage level.
- **Expected impact:** Better citation potential.
- **Implementation guidance:** Keep each section focused on one idea.
- **Example:** One H2 per subtopic with a short explanatory block below.
- **Exceptions:** Very short pages can stay compact.
- **Potential drawbacks:** Excessive chunking can feel mechanical.
- **Conflicting opinions:** None in the report.
- **Notes:** This overlaps with content chunking, but here the focus is extractability.

### 2) Build trust signals through clarity, authorship, and sourcing
- **Rule name:** Trust signals
- **Description:** The report emphasizes citations, original data, and transparent sourcing as trust signals.
- **Source URL:** https://www.lumar.io/blog/best-practice/4-pillar-geo-strategy-framework-for-ai-search-visibility/
- **Source organization:** Lumar
- **Source type:** Industry analysis
- **Classification:** Emerging GEO Practice
- **Severity:** Opportunity
- **Confidence level:** Medium
- **Evidence strength:** Moderate
- **Why it matters:** AI systems and users may prefer content that looks authoritative and verifiable.
- **Expected impact:** Better perceived reliability.
- **Implementation guidance:** Use bylines, organization pages, and source references where relevant.
- **Example:** An author page and organization page linked from the article.
- **Exceptions:** Anonymous or purely creative content is different.
- **Potential drawbacks:** Requires editorial consistency.
- **Conflicting opinions:** None in the report.
- **Notes:** The report does not turn these into formal Google ranking rules.

### 3) Keep entity names explicit and consistent
- **Rule name:** Entity consistency
- **Description:** Use the same entity names consistently across the page and site.
- **Source URL:** https://www.lumar.io/blog/best-practice/4-pillar-geo-strategy-framework-for-ai-search-visibility/
- **Source organization:** Lumar
- **Source type:** Industry analysis
- **Classification:** Emerging GEO Practice
- **Severity:** Opportunity
- **Confidence level:** Medium
- **Evidence strength:** Moderate
- **Why it matters:** Entity clarity helps both AI retrieval and knowledge graph alignment.
- **Expected impact:** Better entity recognition.
- **Implementation guidance:** Keep names, terms, and schema aligned.
- **Example:** Use the exact product name consistently.
- **Exceptions:** Synonyms are fine when natural, but not at the cost of clarity.
- **Potential drawbacks:** Repetition can be awkward if overdone.
- **Conflicting opinions:** None in the report.
- **Notes:** This rule is central to the report’s GEO framing.
